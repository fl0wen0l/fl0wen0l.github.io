Blog 1

Dự án đề xuất: Serverless Photo Analytics & Moderation

Mục tiêu

Xây dựng ứng dụng album ảnh serverless cho phép người dùng đăng nhập, tải ảnh, tự động gắn nhãn (labels) và kiểm duyệt nội dung, tìm kiếm theo nhãn, phân phối toàn cầu qua CDN, kèm dashboard phân tích sử dụng và cảnh báo chi phí. Dự án tối ưu để 5 vai trò: 2 Front-end, 1 Back-end, 1 Security, 1 Research.

Giá trị nổi bật (điểm cộng)

  • Kiến trúc cloud-native, không quản lý server, mở rộng tốt, chi phí thấp.
  • Trải rộng 5 trụ cột Well-Architected: Security, Reliability, Performance, Cost, Operational Excellence.
  • Có CI/CD, IaC, quan sát (observability) đầy đủ để demo chuyên nghiệp.

Kiến trúc mục tiêu (AWS)

  • Giao diện: S3 static website + CloudFront (HTTPS, cache, edge). Optional A/B bằng CloudFront Functions.
  • AuthN/AuthZ: Amazon Cognito (User Pool, Hosted UI, Group/Role mapping Basic vs Admin).
  • API: Amazon API Gateway (REST) hoặc AWS AppSync (GraphQL). Đề xuất AppSync để có realtime subscription cho moderation log.
  • Compute: AWS Lambda (Node.js/Python) cho các nghiệp vụ.
  • Lưu trữ:
    • Ảnh gốc: Amazon S3 (encryption SSE-KMS, block public access, presigned URL upload).
    • Metadata: Amazon DynamoDB (bảng Photos, Labels, Moderation queue). Secondary indexes để truy vấn theo user, label, trạng thái.
  • Xử lý sự kiện:
    • S3 Event → SQS (decouple) → Lambda xử lý (Rekognition DetectLabels + ModerationLabels).
    • Step Functions cho workflow: ghi metadata → nhận diện → cập nhật trạng thái → notify.
  • Tìm kiếm nâng cao (tùy chọn): Amazon OpenSearch Serverless cho search theo nhiều trường/label.
  • Quan sát: CloudWatch Logs, Metrics, Alarms, X-Ray trace; AppConfig/Parameter Store cho cấu hình runtime.
  • Phân tích/BI:
    • Log sự kiện (view, search, upload) gửi về Kinesis Firehose → S3 (data lake). Glue Crawler + Athena để truy vấn. QuickSight dựng dashboard.
  • Bảo mật:
    • AWS WAF gắn vào CloudFront; Shield Standard; IAM least-privilege; S3 bucket policy chặt chẽ; Secrets Manager/KMS.
  • CI/CD & IaC:
    • IaC: AWS CDK (TypeScript) hoặc Terraform (chọn CDK để đồng bộ AWS). Monorepo FE+BE+IaC.
    • CI/CD: CodePipeline (Source GitHub/CodeCommit → CodeBuild → CDK deploy) hoặc GitHub Actions + CDK.

Luồng chính

  • User đăng nhập Cognito → Frontend nhận ID token.
  • FE yêu cầu presigned URL → tải ảnh lên S3.
  • S3 phát sự kiện → SQS → Lambda xử lý: Rekognition labels + moderation; ghi DynamoDB.
  • Ảnh bị nghi ngờ → trạng thái “PENDING_REVIEW”. Admin UI nhận realtime event (AppSync subscription) để duyệt.
  • FE Gallery truy vấn ảnh ĐÃ duyệt; tìm theo label; phân phối qua CloudFront (signed cookies/URL nếu cần).
  • Các sự kiện sử dụng được stream vào S3 → Athena/QuickSight hiển thị dashboard.

Phân công & Deliverables

  • FE 1 (Gallery Web):
    • SPA: Đăng nhập, upload (presigned URL), gallery, filter theo labels, infinite scroll.
    • Tối ưu: lazy-load, responsive, CloudFront cache control.
    • Acceptance: Lighthouse Performance ≥ 85, chức năng hoạt động end-to-end.
  • FE 2 (Admin Moderation):
    • UI duyệt ảnh: xem labels/moderation score, approve/reject, tag editor.
    • Realtime feed (AppSync subscriptions) và audit log.
    • Acceptance: Thao tác < 2s phản hồi, pagination, audit trail.
  • Back-end:
    • AppSync schema/resolvers hoặc API Gateway routes; Lambda functions; Step Functions; SQS; DynamoDB schema (GSIs); presigned URL service.
    • Observability: structured logs, X-Ray, CloudWatch dashboards; error budget SLO.
    • Acceptance: 90% unit test BE, p95 API < 300ms (cache hit), IaC deploy xanh.
  • Security:
    • IAM least-privilege; S3/KMS; WAF rules (rate-based, IP allow/deny, bot control basic); Cognito password policy + email OTP; Config rules + Security Hub findings triage; Budgets cảnh báo chi phí.
    • Pen-test cơ bản (OWASP ASVS check), SRR báo cáo Well-Architected (Security pillar).
    • Acceptance: Scan không có High/Critical, WAF chặn demo payload, báo cáo tuân thủ.
  • Research:
    • Thiết kế A/B (nút upload nổi vs cố định) tại edge; thu thập metric; phân tích Athena/QuickSight; đề xuất cải tiến cost/perf.
    • Benchmark Rekognition thresholds, chọn cut-off phù hợp; thử OpenSearch Serverless vs DynamoDB GSI cho search.
    • Acceptance: Báo cáo thực nghiệm có số liệu, kết luận rõ ràng, PRD cải tiến.

Tiêu chí chấm điểm gợi ý (rubric)

  • Kiến trúc & IaC (25%): Sơ đồ rõ, CDK/Terraform repeatable, multi-env.
  • Bảo mật (20%): IAM least-privilege, KMS, WAF, private-by-default, audit.
  • Chất lượng sản phẩm (20%): UX mượt, error handling, accessibility cơ bản.
  • Vận hành & Quan sát (15%): Logs/metrics/traces, alarms, runbook.
  • Hiệu năng & Chi phí (10%): Cache hit rate, p95 latency, cảnh báo ngân sách.
  • Trình bày & Tài liệu (10%): README, sơ đồ, demo script, kết quả A/B.

Demo script 6-8 phút

  • Sơ đồ kiến trúc (1’). 2) Đăng nhập, upload ảnh (1’). 3) Tự dán nhãn + moderation (1.5’).
  • Admin duyệt realtime (1’). 5) Tìm kiếm theo label, CDN toàn cầu (1’).
  • Dashboard Athena/QuickSight + cảnh báo Budgets (1’). 7) WAF rule chặn payload mẫu (30s).

Lộ trình 3-4 tuần

  • Tuần 1: IaC khung (VPC optional), Cognito, S3, CloudFront, AppSync/API, DynamoDB; FE skeleton; pipeline CI/CD.
  • Tuần 2: Upload flow + presigned URL; S3→SQS→Lambda→Rekognition→DynamoDB; Gallery cơ bản; logs/dashboards.
  • Tuần 3: Admin moderation + subscriptions; WAF + bảo mật; data lake sự kiện (Firehose→S3→Glue→Athena); Budgets.
  • Tuần 4 (buffer): QuickSight, tối ưu hiệu năng, A/B test, báo cáo & rehearsal demo.

Stretch goals (tùy chọn ăn điểm)

  • Textract OCR để tìm kiếm văn bản trong ảnh.
  • Signed URL/Signed Cookies cho ảnh riêng tư; PWA offline.
  • OpenSearch Serverless cho full-text/aggregations; gợi ý label bằng Bedrock Titan (nếu được cấp quyền).
  • Multi-tenant (namespace theo Cognito sub/tenant id).

Chi phí ước lượng (thấp)

  • Free-tier thân thiện: S3, DynamoDB (on-demand thấp), Lambda, CloudFront mức nhỏ; Rekognition trả theo ảnh (giới hạn dataset demo). Thiết lập Budget cảnh báo 5-10 USD.

Tài liệu đi kèm sẽ tạo trong repo

  • Sơ đồ kiến trúc (draw.io/png), README, runbook (sự cố thường gặp), policy-as-code mẫu (cfn-guard/OPA), checklist bảo mật.
